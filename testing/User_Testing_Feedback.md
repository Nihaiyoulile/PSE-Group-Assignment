I.Basic Information of the Test
Test time: Feb 28
Test location: School discussion room
Testers: 3 non-group students
Test method: Task-based usability testing

II.List of Test Tasks
| Number | Task Description | Role |
|--------|-----------------|------|
| T-01 | Find the Database Systems course and view the materials according to different styles. | Student |
| T-02 | Ask a question on the course page | Student |
| T-03 | Upload a material for any course | Lecturer |
| T-04 | As a lecturer, answer students' questions | Lecturer |
| T-05 | As an administrator, approve an account and block an account | Administrator |
| T-06 | Approve or reject a course | Administrator |

III.Feedback Record

After testing, all three testers gave feedback in the T-01 task. The software did not display the style of the material, so they could not make a choice. After that, all the tasks from T-02 to T-05 were normal. However, in the T-06 task, three testers gave feedback that there was no page to approve and reject the course, so the task could not be completed.

IV.Summary of Positive Feedback
1.The UI design style is unified, using minimalist style, and the style is very distinctive.
2.The information is clearly displayed, and the operation is very simple and convenient.
3.In the administrator interface, the number of Pending and Course Requests above will change with the operation, which is very detailed.

V.Summary of Problems
| Question number | Describe | Severity |
|----------------|----------|----------|
| P-01 | The software did not display the style of the material | Medium |
| P-02 | Lack of the function of approving courses | High |

VI.Improvement Measures
| Number | Improvement plan | Status |
|--------|-----------------|--------|
| P-01 | Divide the materials into academic style or casual style, and list them separately. | Finished |
| P-02 | Add the function that the administrator can approve or reject the course. | Finished |

VII.Summary
Most of the test goals have been completed, but some problems need to be improved. The most important finding in the test is that the software lacks many functions. However, it has also won the unanimous affirmation of testers in some aspects, such as UI design. This test is of great significance to the prototype, which not only helps the prototype to conform to the design of the UML diagram, but also obtains the real opinions of users.This gives us a direction in the subsequent improvement of the software, and knows our advantages and disadvantages, which helps us improve ourselves.
