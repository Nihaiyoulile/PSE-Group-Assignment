I.Software Overview

The result of this project is a software prototype system—Campus Connect. This software is aimed at three types of users on the university campus: Student, lecturer and administrator. At the same time, the software completes the functions of each role according to the requirements of use case diagram, class diagram and other UML diagrams and User story.

II.Summary of the Functions of Each Role

| Role | Function |
|------|----------|
| Student | Search courses, view materials, ask questions, add bookmarks, report content |
| Lecturer | Add courses, upload materials, and answer questions |
| Administrator | Approve or block the account, content review, course approval |

Compared with User Story in Product Backlog, the final version of the software prototype has completed almost all the content.

III.Software Limitations

1. There is no back-end, the function is only for interface demonstration, no data storage
2. The data will not be retained after the page is refreshed. 
3. The learning materials in the Student module are all demonstrations and cannot be opened. In the Lecturer module, the editing course is also a demonstration and cannot be opened.

IV.Future Improvement Direction

1. Connect to the real database to ensure that the data can be stored for a long time 
2. Add the email notification function 
3. Add the function of uploading and downloading real files

V.Summary

The overall completion of the software is relatively high, achieving the original goal of Product Vision.It is for campus teaching scenarios, which is used to integrate course materials, learn questions and answers and role-based management functions.This supports more efficient teaching and learning interaction, and at the same time echoes the target requirements of SDG4 for high-quality education.
